## Design Context

### Users

- 主要用户是使用 Gemini 生成图片并需要去水印的普通用户。
- 他们通常希望快速完成一件具体的小事，不想学习复杂流程，也不想面对“专业工具”式的门槛。
- 他们的核心任务是：上传图片，快速得到可下载的去水印结果，并且明确知道图片始终只在本地浏览器中处理。

### Brand Personality

- 品牌关键词：简单、易用、友好。
- 情绪目标：轻松愉快，而不是严肃、技术化或压迫感强。
- 产品语气应该直接、清楚、温和，避免营销化、夸张化和“AI 套壳产品”的话术。

### Aesthetic Direction

- 视觉方向采用浅色模式，围绕 `🍌`、香蕉黄和奶油白展开，但整体应保持“轻量工具”而不是产品介绍页。
- 组件层统一建立在 `shadcn/ui + Radix` 的轻量设计系统上，颜色、圆角、边框、焦点态和弹层都要走统一 token，不再堆一次性样式。
- 可以保留少量品牌细节，但不要堆装饰、不要放大叙事内容，也不要让页面像落地页。
- 交互上优先强调清晰反馈、低学习成本和过程可感知；动画只服务于状态切换、上传反馈、hover 提示和预览切换。

### Design Principles

- 单一任务优先：所有界面元素都要服务于“上传、处理、预览、下载”这一条主路径。
- 说明克制：默认只保留一两句必要说明，避免 FAQ、导览、卖点区块挤占主界面。
- 轻松但不轻浮：保持友好和轻快，同时让状态反馈、下载动作和结果展示足够可靠。
- 像产品，不像工具站：减少噪音、减少功能堆叠、减少信息密度，保留清晰层级和舒适节奏。
- 默认简单：不给用户多余决策，重要操作直接可见，复杂度只在必要时逐步显露。
- 设计系统优先：优先复用统一组件、语义化颜色和稳定交互模式，避免重复造按钮、选择器和弹窗。
- 首屏自解释：上传方式、本地处理承诺和结果流转要在第一屏说清楚，降低第一次使用的理解成本。
- 浅色一致性：后续设计默认只做浅色模式，颜色、阴影、对比和插画都围绕这一前提统一。
